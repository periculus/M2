# M2 Code Intelligence Features
Last verified: 2026-02-10 | M2 1.25.06 | JupyterLab 4.4.5 | Node 25.1.0 | Lezer 1.4.x
Source of truth for: hover, completion, folding, indentation, extending features

## Overview

All code intelligence is driven by `src/m2Symbols.json` (266 KB, 1763 entries, 100% documented). The extension provides: hover tooltips, autocompletion, context-aware option completion, code folding, and auto-indentation.

## Symbol Database Schema (`src/m2Symbols.json`)

Each entry:
```json
{
  "label": "gb",              // Symbol name (required)
  "type": "function",         // Category: "keyword" | "type" | "function" | "constant"
  "info": "compute a Gröbner basis",     // Headline description (100% coverage)
  "detail": "gb I",           // Usage pattern (718 have this)
  "options": [                // Function options (82 functions have these)
    {"name": "Strategy", "type": "Symbol", "info": "algorithm choice"}
  ],
  "inputs": [                 // Input parameters (527 have these)
    {"name": "I", "type": "Ideal, Module, or Matrix"}
  ],
  "outputs": [                // Output types (406 have these)
    {"type": "GroebnerBasis"}
  ]
}
```

Generated by `scripts/generate_symbols.py`. See [symbol-pipeline.md](symbol-pipeline.md) for regeneration.

## Hover Tooltips (`src/m2Hover.ts`)

**How it works:**
1. `hoverTooltip()` callback fires on mouse hover
2. Resolves cursor position to a Lezer tree node
3. Only responds to: `Identifier`, `Type`, `Builtin`, `Constant`, `Boolean`
4. Looks up the word in `symbolMap` (O(1) hash map built at load)
5. Builds DOM tooltip with sections:

```
┌──────────────────────────────────────┐
│ gb  function                         │  ← header + badge
│ compute a Gröbner basis              │  ← info (headline)
│ gb I                                 │  ← detail (usage, code-styled)
│ Input: I — Ideal, Module, or Matrix  │  ← inputs section
│ Output: GroebnerBasis                │  ← outputs section
│ Options: Strategy, DegreeLimit, ...  │  ← options (max 8 shown)
└──────────────────────────────────────┘
```

**CSS classes** (in `style/index.css`):
- `.m2-hover-tooltip` — container (max-width 400px)
- `.m2-hover-header` — flexbox name + badge
- `.m2-hover-badge-{type}` — colored badge per category
- `.m2-hover-info` — italic headline
- `.m2-hover-usage` — code-styled usage pattern
- `.m2-hover-section` / `.m2-hover-section-label` — input/output/options

## Autocompletion (`src/m2Language.ts`)

### Base Completion
```typescript
const baseCompletion = completeFromList(m2Symbols);
```
All 1763 symbols available everywhere. CodeMirror handles fuzzy matching and ranking.

### Context-Aware Option Completion
```typescript
function m2OptionCompletion(context: CompletionContext): CompletionResult | null
```

**How it works:**
1. Walk Lezer tree upward from cursor to find enclosing `CallExpr`
2. If found, extract function name from `CallExpr.firstChild`
3. Look up in `symbolsWithOptions` map (82 functions)
4. Return function's options as completions with ` => ` suffix, boosted priority

**Example:** Inside `gb(I, |)`, offers `Strategy =>`, `DegreeLimit =>`, etc.

Registered via: `M2Language.data.of({ autocomplete: m2OptionCompletion })`

## Code Folding (`src/m2Language.ts`)

Configured via `foldNodeProp.add()`:

| Node Type | Fold Range | Min Size |
|-----------|-----------|----------|
| `ParenExpr` | `(` to `)` contents | >40 chars |
| `ListExpr` | `{` to `}` contents | >40 chars |
| `ArrayExpr` | `[` to `]` contents | >40 chars |
| `CallExpr` | Opening delim to closing delim | >40 chars (from delim) |
| `BlockComment` | `-*` to `*-` contents | Always |

**CallExpr technique:** Find the opening delimiter by getting `node.firstChild` (the function name), then fold from `firstChild.to + 1` to `node.to - 1`. This skips the function name so it stays visible.

## Auto-Indentation (`src/m2Language.ts`)

Configured via `indentNodeProp.add()` with `continuedIndent()`:
- `ParenExpr`, `ListExpr`, `ArrayExpr` — indent continuation lines
- `IfExpr`, `ForExpr`, `WhileExpr` — indent body

## Language Registration (`src/m2Language.ts`)

```typescript
const M2Language = LRLanguage.define({
  parser: parserWithProps,  // parser + fold + indent props
  languageData: {
    commentTokens: { line: '--', block: { open: '-*', close: '*-' } },
    closeBrackets: { brackets: ['(', '[', '{', '"'] },
    autocomplete: baseCompletion
  }
});
```

Exported as `m2()` returning `LanguageSupport` with:
- `m2EditorAttrs` — sets `data-language="macaulay2"` for CSS scoping
- `syntaxHighlighting(m2ExtraHighlights)` — custom highlight rules
- `m2HoverTooltip` — hover tooltip provider
- `m2OptionCompletion` — context-aware option completion

## How to Add a New Feature

### Signature Help (parameter hints)
Would need: a CodeMirror tooltip that shows function signature when typing inside `CallExpr`. Walk tree to find function name, look up in symbolMap, display inputs with current parameter highlighted.

### Linting
Would need: a CodeMirror `linter()` extension. Could flag: undefined symbols, type mismatches (if types are tracked), or common mistakes.

### Go-to-Definition (in extension, not kernel)
Would need: track symbol definitions from parse tree (assignments like `x := ...`), then implement `EditorView.domEventHandlers` for Ctrl+Click to jump to definition.

## Smoke Test Checklist

After any code intelligence change, verify in JupyterLab:
- [ ] Hover `gb` → shows headline, usage, inputs, outputs, options
- [ ] Hover `ZZ` → shows "the ring of integers" or similar (type)
- [ ] Hover `infinity` → shows constant description
- [ ] Type `gb(I, ` then Tab → shows `Strategy =>`, `DegreeLimit =>` options
- [ ] Type `id` then Tab → shows `ideal` in completions
- [ ] Multi-line `ideal(\n  x^2,\n  y^2\n)` → fold arrow appears on first line
- [ ] Click fold arrow → collapses to `ideal(...)`
- [ ] Press Enter inside `(` → auto-indents
